## Data Flow

```
User
в””в”Җв–ә Seerr (request)
    в””в”Җв–ә Sonarr / Radarr (pick release, apply quality profile)
        в””в”Җв–ә Prowlarr (search indexers)
            в””в”Җв–ә SABnzbd (download + unpack to /mnt/downloads)
                в””в”Җв–ә Sonarr / Radarr (import вҶ’ /mnt/media/tv or /mnt/media/movies)
                    в””в”Җв–ә Plex (detects new file, scans library, ready to stream)
```

## Storage Paths

| Path | Used by | Description |
|------|---------|-------------|
| `/mnt/media/movies` | Radarr, Plex | Final movie library |
| `/mnt/media/tv` | Sonarr, Plex | Final TV show library |
| `/mnt/downloads` | SABnzbd, Sonarr, Radarr | Download staging area |

All three paths come from the same ZFS pool. Sonarr and Radarr see both `/mnt/downloads` and `/mnt/media` вҖ” this is what allows them to use hard links during import instead of copying files.

## Network

All Docker containers run on a shared internal network (`media_net`). They communicate by container name, not by port or IP. The only ports exposed to the LAN are the web UIs of each service.

| Service | LAN Port |
|---------|----------|
| Plex | 32400 |
| Seerr | 5055 |
| Sonarr | 8989 |
| Radarr | 7878 |
| Prowlarr | 9696 |
| SABnzbd | 8080 |

None of these are exposed to the internet by default. Remote access to Plex is handled by Plex's own relay service.
